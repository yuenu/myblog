React如何實現Vue的keep-alive功能？ 初探 React Router v6

 2023/01/17

專案上需要用到 cache 頁面

最近剛好有個需求是在某些頁面的情況之下，

使用者從 A 畫面 -> 跳至 B 畫面 -> 再回到 A 畫面時，

scroll 的狀態要維持在原本的位置，

有玩過 Vue 的人就知道這用Vue.js的 keep-alive 很容易就可以實現了，

然後此專案是用 React class component 寫的，所以選擇不多，最後選擇用react-router-cache-route來實現此功能

它的原理就是在要 cache 的這個頁面外層加個 div 然後切換此頁面時在此 div 設置 display: none;

切換到 cache 的頁面時再 display: block;就行了

思路很簡單，不過如果要 cache 的頁面越來越多的情況下，html 的結構會越長越大，這實在不是個好辦法

Vue.js 的 keep-alive

源碼在此

之後開始研究了 Vue.js 的 keep-alive 是如何實現的，

而 keep-alive 是透過LRU cache策略儲存在 VNode

但因此就讓我思考了難道 React 沒有如 keep-alive cache 頁面的方法

目前查到 2 個方法

  • React 18 OffScreen

  • 基于 react-router V6 实现 路由缓存
